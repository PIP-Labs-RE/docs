DATA Foundation Gateway(SPG)는 우리 프로토콜에 배포된 주변/유틸리티 스마트 컨트랙트 그룹으로, [🧩 IP 자산](/concepts/ip-asset/overview)을 등록하고 해당 IP 자산에 라이선스 약관을 부착하는 것과 같은 **독립적인 작업들을 결합하여 단일 트랜잭션으로 처리할 수 있게 해주어, 개발자의 작업을 더 쉽게 만들어줍니다**.

이는 주로 우리 [SDK](/sdk-reference)를 더 쉽게 사용할 수 있도록 개발되었습니다.

예를 들어, 이 `mintAndRegisterIpAndAttachPILTerms`는 SPG에 있는 함수 중 하나(더 구체적으로는 `LicenseAttachmentWorkflows.sol` 내)로, NFT를 민팅하고, 이를 IP 자산으로 등록하고, 라이선스 약관을 부착하는 작업을 단일 호출로 수행할 수 있게 해줍니다:

```solidity LicenseAttachmentWorkflows.sol
function mintAndRegisterIpAndAttachPILTerms(
  address spgNftContract,
  address recipient,
  WorkflowStructs.IPMetadata calldata ipMetadata,
  WorkflowStructs.LicenseTermsData[] calldata licenseTermsData,
  bool allowDuplicates
) external onlyMintAuthorized(spgNftContract) returns (address ipId, uint256 tokenId, uint256[] memory licenseTermsIds)
```

## 호출 일괄 처리

SPG에는 `mintAndRegisterIpAndAttachPILTerms`, `registerIpAndAttachPILTerms` 등 다양한 함수가 포함되어 있지만, IP 자산과의 가능한 모든 상호작용 조합을 위해 컨트랙트를 지속적으로 업데이트하는 것은 번거로운 일입니다.

대신, 원하는 방식으로 트랜잭션을 일괄 처리할 수 있는 "Multicall" 메커니즘을 허용했습니다. 자세한 내용은 [배치 함수 호출](/concepts/spg/batch-spg-function-calls)을 참고하세요.
