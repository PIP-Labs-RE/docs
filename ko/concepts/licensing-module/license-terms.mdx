DATA Foundation에 IP를 등록할 때, 해당 IP에 라이선스 약관(License Terms)을 부착할 수 있습니다. 이는 [📜 Licensing Module](/concepts/licensing-module/overview)에 의해 온체인에서 강제되는 실질적이고 법적 구속력이 있는 약관이며, 최악의 경우 전통적인 방식으로 법정에서 오프체인으로도 집행될 수 있습니다.

여기에는 상업적 사용에 대한 조건(예: "수익의 50%를 부모 IP와 공유해야 함")도 포함됩니다.

더 구체적으로 말하면, 라이선스 약관은 [License Template](/concepts/licensing-module/license-template)의 값들로 이루어진 특정 조합입니다. 실제로 각 라이선스 템플릿에 대해 **여러 개의** 라이선스 약관(변형)이 존재할 수 있고, 또 존재하게 됩니다. 라이선스 템플릿이 다양한 라이선스 약관 변형을 생성한다고 생각하면 됩니다.

한 번 등록되면, **라이선스 약관은 불변입니다: 변조하거나 변경할 수 없습니다**. 이를 생성한 라이선스 템플릿조차도 변경할 수 없습니다.

또한, 라이선스 약관은 해당 라이선스 템플릿 내에서 고유한 숫자 ID를 가집니다. 이로 인해 라이선스 약관은 재사용 가능합니다. 즉, 누군가가 특정 값 조합으로 라이선스 약관을 생성하면, 단 한 번만 생성하면 되며 다른 누구나 이를 사용할 수 있습니다.

예를 들어, [Programmable IP License (PIL💊)](/concepts/programmable-ip-license/overview)의 특정 약관 값 집합(예: 비상업적 사용 + 파생물 허용 + 무료 민팅)은 고유한 ID를 가지는 하나의 라이선스 약관을 정의합니다.

## IP 자산에 부착된 라이선스 약관

루트 IP 자산의 소유자는 라이선스 약관을 부착하여, 다른 사용자들이 해당 약관의 라이선스 토큰을 민팅해서 이 IP 자산의 파생물을 만들 수 있다고 알릴 수 있습니다. **라이선스 약관이 IP 자산에 부착되면, 그 자산은 이제 "공개" 상태로 간주되며 누구든지 해당 약관을 사용해서 라이선스 토큰을 민팅할 수 있습니다.**

## 상속된 라이선스 약관

반면에, 파생 IP 자산은 부모 IP 자산으로부터 라이선스 약관을 상속받습니다. 이는 IP 자산이 자신을 파생물로 등록할 때, 라이선스 토큰을 소각하고 관련된 라이선스 약관을 상속받는다는 의미입니다. **이 파생물의 소유자는 새로운 라이선스 약관을 설정할 수 없습니다.**

<Note>

이런 의문이 생길 수 있습니다: "내 파생물에 새로운 라이선스 약관을 설정할 수 없다면, 그것은 내 파생물의 민팅 수수료를 변경하거나 추가 파생물을 허용하지 않게 하는 것도 불가능하다는 의미인가요?"

다행히도, 이를 우회하는 방법이 있습니다! 파생 IP의 라이선스 약관은 변경할 수 없지만, [License Config을 활용하여 특수한 동작을 구현](/concepts/licensing-module/license-config)할 수 있습니다.

</Note>

## 만료

라이선스 약관은 `expiration` 시간을 지원합니다. 라이선스 약관이 만료되면, 해당 라이선스를 따르는 모든 파생물은 더 이상 수익을 창출하거나 추가 파생물을 만들 수 없게 됩니다. IP 자산이 여러 부모의 파생물인 경우, 두 부모 중 더 빠른 만료 시간이 도달했을 때 만료됩니다.
